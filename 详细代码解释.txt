   ① JVM加载Main类
   ② 调用main方法

   ④ 创建Environment（环境变量、配置文件）

```java
@PostMapping("/chat")
public ResponseEntity<ApiResponse<ChatResponse>> chat(@RequestBody ChatRequest request) {
    // 1. 参数校验
    if (request.getQuestion() == null || request.getQuestion().trim().isEmpty()) {
        return ResponseEntity.ok(ApiResponse.success(ChatResponse.error("问题内容不能为空")));
    }

    // 2. 获取或创建会话
    SessionInfo session = getOrCreateSession(request.getId());
    List<Map<String, String>> history = session.getHistory();

    // 3. 创建DashScope API和ChatModel
    DashScopeApi dashScopeApi = chatService.createDashScopeApi();
    DashScopeChatModel chatModel = chatService.createStandardChatModel(dashScopeApi);

    // 4. 构建系统提示词（包含历史消息）
    String systemPrompt = chatService.buildSystemPrompt(history);
    
    // 5. 创建ReactAgent
    ReactAgent agent = chatService.createReactAgent(chatModel, systemPrompt);
    
    // 6. 执行对话
    String fullAnswer = chatService.executeChat(agent, request.getQuestion());
    
    // 7. 更新会话历史
    session.addMessage(request.getQuestion(), fullAnswer);
    
    return ResponseEntity.ok(ApiResponse.success(ChatResponse.success(fullAnswer)));
}
```

解释:

- ResponseEntity<ApiResponse<ChatResponse>>: 三层包装

  * ApiResponse: 统一响应格式（code、message、data）
  * ChatResponse: 对话响应（success、answer、errorMessage）
- 执行流程:

2.3 流式对话接口
```java
@PostMapping(value = "/chat_stream", produces = "text/event-stream;charset=UTF-8")
public SseEmitter chatStream(@RequestBody ChatRequest request) {
    SseEmitter emitter = new SseEmitter(300000L); // 5分钟超时

    executor.execute(() -> {
        try {
            // ... 创建Agent同上 ...
            
            StringBuilder fullAnswerBuilder = new StringBuilder();
            
            // 使用agent.stream()进行流式对话
            Flux<NodeOutput> stream = agent.stream(request.getQuestion());
            
            stream.subscribe(
                output -> {
                    if (output instanceof StreamingOutput streamingOutput) {
                        OutputType type = streamingOutput.getOutputType();
                        
                        if (type == OutputType.AGENT_MODEL_STREAMING) {
                            String chunk = streamingOutput.message().getText();
                            if (chunk != null && !chunk.isEmpty()) {
                                fullAnswerBuilder.append(chunk);
                                emitter.send(SseEmitter.event()
                                        .name("message")
                                        .data(SseMessage.content(chunk), MediaType.APPLICATION_JSON));
                            }
                        }
                    }
                },
                error -> { /* 错误处理 */ },
                () -> {
                    // 完成处理
                    String fullAnswer = fullAnswerBuilder.toString();
                    session.addMessage(request.getQuestion(), fullAnswer);
                    emitter.send(SseEmitter.event()
                            .name("message")
                            .data(SseMessage.done(), MediaType.APPLICATION_JSON));
                    emitter.complete();
                }
            );
        } catch (Exception e) {
            emitter.completeWithError(e);
        }
    });

    return emitter;
}
```

解释:
- produces = "text/event-stream": 指定响应类型为SSE（Server-Sent Events）
- SseEmitter: Spring提供的SSE发射器，支持异步推送
- executor.execute(): 在线程池中异步执行，避免阻塞Servlet线程
- agent.stream(): 返回Flux响应式流，逐步输出Agent的思考过程
- stream.subscribe(): 订阅流的三个回调：
  * onNext: 每收到一个数据块就推送到前端
  * onError: 发生错误时推送错误信息
  * onComplete: 流结束时推送done标记并关闭emitter
- AGENT_MODEL_STREAMING: Agent正在生成文本，逐字推送
- AGENT_TOOL_FINISHED: 工具调用完成，记录日志
- 优势: 用户可以实时看到AI思考过程，体验更好

2.4 AIOps接口
```java
@PostMapping(value = "/ai_ops", produces = "text/event-stream;charset=UTF-8")
public SseEmitter aiOps() {
    SseEmitter emitter = new SseEmitter(600000L); // 10分钟超时

    executor.execute(() -> {
        try {
            DashScopeApi dashScopeApi = chatService.createDashScopeApi();
            DashScopeChatModel chatModel = DashScopeChatModel.builder()
                    .dashScopeApi(dashScopeApi)
                    .defaultOptions(DashScopeChatOptions.builder()
                            .withModel(DashScopeChatModel.DEFAULT_MODEL_NAME)
                            .withTemperature(0.3)  // 更低温度，更确定性
                            .withMaxToken(8000)    // 更大token，支持长报告
                            .withTopP(0.9)
                            .build())
                    .build();

            ToolCallback[] toolCallbacks = tools.getToolCallbacks();

            emitter.send(SseEmitter.event().name("message")
                    .data(SseMessage.content("正在读取告警并拆解任务...\n")));
            
            // 执行AIOps分析
            Optional<OverAllState> overAllStateOptional = 
                aiOpsService.executeAiOpsAnalysis(chatModel, toolCallbacks);

            if (overAllStateOptional.isEmpty()) {
                emitter.send(SseEmitter.event().name("message")
                        .data(SseMessage.error("多Agent编排未获取到有效结果")));
                emitter.complete();
                return;
            }

            OverAllState state = overAllStateOptional.get();
            
            // 提取最终报告
            Optional<String> finalReportOptional = aiOpsService.extractFinalReport(state);

            if (finalReportOptional.isPresent()) {
                String finalReportText = finalReportOptional.get();
                
                // 流式输出报告（每次50字符）
                int chunkSize = 50;
                for (int i = 0; i < finalReportText.length(); i += chunkSize) {
                    int end = Math.min(i + chunkSize, finalReportText.length());
                    String chunk = finalReportText.substring(i, end);
                    emitter.send(SseEmitter.event().name("message")
                            .data(SseMessage.content(chunk), MediaType.APPLICATION_JSON));
                }
            }

            emitter.send(SseEmitter.event().name("message")
                    .data(SseMessage.done(), MediaType.APPLICATION_JSON));
            emitter.complete();
        } catch (Exception e) {
            emitter.completeWithError(e);
        }
    });

    return emitter;
}
```

解释:
- 无参数接口，自动触发AIOps流程
- 超时时间10分钟（600000ms），因为多Agent协作可能较慢
- Temperature=0.3: 降低随机性，使输出更稳定
- MaxToken=8000: 允许生成长篇报告
- executeAiOpsAnalysis(): 调用AiOpsService执行多Agent协作
- extractFinalReport(): 从state中提取planner_plan的最终报告
- 流式输出: 将报告按50字符分块逐步推送，提升用户体验

2.5 会话管理内部类
```java
private static class SessionInfo {
    private final String sessionId;
    private final List<Map<String, String>> messageHistory;
    private final long createTime;
    private final ReentrantLock lock;

    public void addMessage(String userQuestion, String aiAnswer) {
        lock.lock();
        try {
            // 添加用户消息
            Map<String, String> userMsg = new HashMap<>();
            userMsg.put("role", "user");
            userMsg.put("content", userQuestion);
            messageHistory.add(userMsg);

            // 添加AI回复
            Map<String, String> assistantMsg = new HashMap<>();
            assistantMsg.put("role", "assistant");
            assistantMsg.put("content", aiAnswer);
            messageHistory.add(assistantMsg);

            // 自动清理：保持最多6对消息
            int maxMessages = MAX_WINDOW_SIZE * 2;
            while (messageHistory.size() > maxMessages) {
                messageHistory.remove(0); // 删除最旧的用户消息
                if (!messageHistory.isEmpty()) {
                    messageHistory.remove(0); // 删除对应的AI回复
                }
            }
        } finally {
            lock.unlock();
        }
    }

    public List<Map<String, String>> getHistory() {
        lock.lock();
        try {
            return new ArrayList<>(messageHistory); // 返回副本
        } finally {
            lock.unlock();
        }
    }
}
```

解释:
- static class: 静态内部类，不持有外部类引用，节省内存
- ReentrantLock: 可重入锁，保证线程安全
- addMessage(): 
  * 加锁保护，防止并发修改
  * 成对添加用户消息和AI回复
  * 滑动窗口：超过6对时删除最旧的2条（一对）
- getHistory():
  * 返回副本而非原列表，防止外部修改影响内部状态
  * 加锁保证读取一致性
- 设计优点:
  * 线程安全：多线程访问同一会话不会冲突
  * 内存控制：限制历史消息数量，防止内存泄漏
  * 数据隔离：返回副本，避免并发修改异常

【关键技术点】
1. SSE流式输出: SseEmitter + ExecutorService异步处理
2. 会话管理: ConcurrentHashMap + ReentrantLock
3. ReactAgent: 自动工具调用，无需手动编排
4. 响应式编程: Flux流式处理Agent输出
5. 统一响应格式: ApiResponse封装所有接口返回

```java
@PostMapping(value = "/api/upload", consumes = "multipart/form-data")
public ResponseEntity<?> upload(@RequestParam("file") MultipartFile file) {
    // 1. 校验文件非空
    if (file.isEmpty()) {
        return ResponseEntity.badRequest().body("文件不能为空");
    }

    String originalFilename = file.getOriginalFilename();
    if (originalFilename == null || originalFilename.isEmpty()) {
        return ResponseEntity.badRequest().body("文件名不能为空");
    }

    // 2. 校验文件扩展名
    String fileExtension = getFileExtension(originalFilename);
    if (!isAllowedExtension(fileExtension)) {
        return ResponseEntity.status(HttpStatus.BAD_REQUEST)
                .body("不支持的文件格式，仅支持: " + fileUploadConfig.getAllowedExtensions());
    }

    try {
        String uploadPath = fileUploadConfig.getPath();
        Path uploadDir = Paths.get(uploadPath).normalize();
        if (!Files.exists(uploadDir)) {
            Files.createDirectories(uploadDir);
        }

        // 3. 使用原始文件名（实现基于文件名的去重）
        Path filePath = uploadDir.resolve(originalFilename).normalize();
        
        // 4. 如果文件已存在，先删除旧文件（覆盖更新）
        if (Files.exists(filePath)) {
            logger.info("文件已存在，将覆盖: {}", filePath);
            Files.delete(filePath);
        }
        
        Files.copy(file.getInputStream(), filePath);

        // 5. 文件上传成功后，自动调用向量索引服务
        try {
            logger.info("开始为上传文件创建向量索引: {}", filePath);
            vectorIndexService.indexSingleFile(filePath.toString());
            logger.info("向量索引创建成功: {}", filePath);
        } catch (Exception e) {
            logger.error("向量索引创建失败: {}, 错误: {}", filePath, e.getMessage(), e);
            // 即使索引失败，文件上传仍然成功
        }

        FileUploadRes response = new FileUploadRes(
                originalFilename,
                filePath.toString(),
                file.getSize()
        );

        ApiResponse<FileUploadRes> apiResponse = new ApiResponse<>();
        apiResponse.setCode(200);
        apiResponse.setMessage("success");
        apiResponse.setData(response);
        
        return ResponseEntity.ok(apiResponse);

    } catch (IOException e) {
        ApiResponse<String> errorResponse = new ApiResponse<>();
        errorResponse.setCode(500);
        errorResponse.setMessage("文件上传失败: " + e.getMessage());
        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body(errorResponse);
    }
}
```

解释:
- consumes = "multipart/form-data": 指定接收multipart格式（文件上传）
- @RequestParam("file"): 从表单字段"file"中获取MultipartFile
- 校验流程:
  ① 检查文件是否为空
  ② 检查文件名是否有效
  ③ 检查文件扩展名是否在白名单中（txt, md）
- 文件保存:
  ① 从配置获取上传路径（./uploads）
  ② 创建目录（如果不存在）
  ③ normalize(): 规范化路径，防止路径遍历攻击
  ④ 如果同名文件存在，先删除（实现覆盖）
  ⑤ 复制文件流到目标路径
- 自动索引:
  ① 调用vectorIndexService.indexSingleFile()
  ② 如果索引失败，只记录日志，不影响文件上传
  ③ 这样设计是为了保证文件上传的可靠性
- 响应格式: 统一使用ApiResponse封装

3.2 辅助方法
```java
private String getFileExtension(String filename) {
    int lastIndexOf = filename.lastIndexOf(".");
    if (lastIndexOf == -1) {
        return "";
    }
    return filename.substring(lastIndexOf + 1).toLowerCase();
}

private boolean isAllowedExtension(String extension) {
    String allowedExtensions = fileUploadConfig.getAllowedExtensions();
    if (allowedExtensions == null || allowedExtensions.isEmpty()) {
        return false;
    }
    List<String> allowedList = Arrays.asList(allowedExtensions.split(","));
    return allowedList.contains(extension.toLowerCase());
}
```

解释:
- getFileExtension(): 提取文件扩展名（如"txt"、"md"）
- isAllowedExtension(): 检查扩展名是否在配置允许的列表中
- toLowerCase(): 统一转小写，避免大小写问题

【安全考虑】
1. 路径遍历防护: Paths.get().normalize()防止../../../etc/passwd攻击
2. 文件类型白名单: 只允许txt和md文件
3. 文件大小限制: 需在application.yml中配置spring.servlet.multipart.max-file-size
4. 异常隔离: 索引失败不影响文件上传

6.5 Supervisor提示词
```java
private String buildSupervisorSystemPrompt() {
    return """
            你是AI Ops Supervisor，负责调度planner_agent与executor_agent：
            1. 当需要拆解任务或重新制定策略时，调用planner_agent。
            2. 当planner_agent输出decision=EXECUTE时，调用executor_agent执行第一步。
            3. 根据executor_agent的反馈，评估是否需要再次调用planner_agent，
               直到decision=FINISH。
            4. FINISH后，确保向最终用户输出完整的《告警分析报告》。
            5. 若步骤涉及腾讯云日志/主题工具，请确保使用连字符区域ID（ap-guangzhou等），
               或省略region以采用默认值。
            6. 如果发现Planner/Executor在同一方向连续3次调用工具仍失败或没有数据，
               必须终止流程，直接输出"任务无法完成"的报告，明确告知失败原因，
               严禁凭空编造结果。

            只允许在planner_agent、executor_agent与FINISH之间做出选择。
            """;
}
```

解释:
- Supervisor决策逻辑:
  * 需要规划 → 调用Planner
  * Planner说EXECUTE → 调用Executor
  * Executor返回反馈 → 回到Planner重新评估
  * Planner说FINISH → 结束流程
- 终止条件:
  * decision=FINISH
  * 连续3次失败
- 约束: 只能选择planner_agent、executor_agent、FINISH三者之一

【多Agent协作流程】
```
用户触发AIOps
    ↓
Supervisor启动
    ↓
┌──────────────────────┐
│ 调用Planner          │
│ - 读取告警           │
│ - 拆解任务           │
│ - 输出decision       │
└──────┬───────────────┘
       │
       ├─ decision=PLAN ──→ 继续调用Planner
       │
       ├─ decision=EXECUTE
       │       ↓
       │  ┌────────────────────┐
       │  │ 调用Executor       │
       │  │ - 执行第一步       │
       │  │ - 收集证据         │
       │  │ - 返回feedback     │
       │  └──────┬─────────────┘
       │         │
       │         └─→ 回到Planner（带上feedback）
       │
       └─ decision=FINISH
               ↓
        输出最终报告
```

【设计亮点】
1. 职责分离: Planner思考，Executor行动，Supervisor协调
2. 容错机制: 失败3次自动终止，防止死循环
3. 数据共享: 通过OverAllState传递planner_plan和executor_feedback
4. 结构化输出: 强制Markdown模板，保证报告质量

总结
================================================================================

本项目采用典型的Spring Boot分层架构：

1. Controller层: 接收HTTP请求，参数校验，响应封装
2. Service层: 业务逻辑处理，事务管理
3. Tool层: Agent可调用的工具方法
4. Config层: Bean配置，属性绑定
5. Client层: 外部服务客户端
6. DTO层: 数据传输对象

核心技术：
- Spring AI: ReactAgent智能体
- Milvus: 向量数据库
- DashScope: 阿里云通义千问LLM
- SSE: 服务器推送事件
- ReAct模式: Reasoning + Acting

项目特色：
- 智能化: Agent自动调用工具
- 模块化: 清晰的分层架构
- 可扩展: 新增工具只需添加@Component
- 灵活性: Mock/真实模式切换
- 高性能: SSE流式输出，Milvus向量检索
